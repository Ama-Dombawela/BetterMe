Feedback Page validation report

Team Page validation report

The Team page validation highlighted the need for better use of semantic HTML elements, such as incorporating section tags. The report also identified issues with improperly opened and closed tags, which were corrected to ensure proper HTML structure. Additionally, there were problems with linking CSS,HTML files and images due to spaces in file names. These were resolved by replacing spaces with %20 in the file paths. Some color contrast issues were also flagged and addressed by adjusting the color scheme for better readability. After making these fixes, the page passed validation. This experience emphasized the importance of proper content structure, clean code, and accessibility in creating a user-friendly, optimized webpage

Sitemap Page validation report

The Sitemap page's validation report was mostly clean, with minor issues related to SVG accessibility, focus styles, file path encoding, and improper title tag placement. I added aria-label attributes to SVG icons for better accessibility and ensured all interactive elements had proper focus styles for keyboard navigation. I also corrected %20 encoding issues in file paths to prevent broken links. Additionally, the title tag, which was incorrectly placed inside a rect tag, was moved to the alone title tag element for proper structure. These fixes improved accessibility, usability, and compliance with best practices.
